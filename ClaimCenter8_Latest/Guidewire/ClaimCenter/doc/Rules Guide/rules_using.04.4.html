ClaimCenter Rules Guide : Gosu Business Rules : Rules: A Background : Overview of ClaimCenter Rule Sets

Overview of ClaimCenter Rule Sets

A rule set can be thought of as a logical grouping of rules that are specific to a business function within ­ClaimCenter. Guidewire provides the following ClaimCenter sample rule sets:
 
Rule set category
Contains rules to
Approval Routing Rule Set Category
Route approval activities for a transaction that requires approval to the appropriate user. See “Approval Routing Rule Set Category”.
Archive Rule Set Category
Manage the claim archive process. See “Archive Rule Set Category”.
Assignment Rule Set Category
Determine the responsible party for an activity, claim, exposure, or matter. See “Assignment Rule Set Category”.
BulkInvoice Approval Rule Set Category
Define the approval requirements for Bulk Invoices. See “BulkInvoice Approval Rule Set Category”.
Closed Rule Set Category
Perform an action if you close an activity, claim, exposure, or matter—for example, sending a notification. See “Closed Rule Set Category”.
Event Message Rule Set Category
Handle communication with integrated external applications. See the “Messaging and Events” in the Integration Guidefor information about events and event messaging. See “Event Message Rule Set Category”.
Exception Rule Set Category
Specify an action to take under certain circumstances. For example, if no one touches a claim for a certain number of days, then create a review activity. See “Exception Rule Set Category”.
Initial Reserve Rule Set Category
Determine the initial financial reserves to create for a new exposure. See “Initial Reserve Rule Set Category”.
Loaded Rule Set Category
Trigger automatic actions if the FNOL import interface loads a new claim. See “Loaded Rule Set Category”.
Postsetup Rule Set Category
Trigger automatic actions any time that a new claim is added to ClaimCenter. These rules run after the Assignment and Workplan rules complete their execution. See “Postsetup Rule Set Category”.
Presetup Rule Set Category
Trigger automatic actions any time that a new claim, exposure, matter or transaction is added to ClaimCenter. These rules run prior to the execution of the Segmentation Rule Set, Assignment Rule Set, and Workplan Rule Set rules. See “Presetup Rule Set Category”.
Preupdate Rule Set Category
Trigger automatic actions any time that a claim or other high-level entity changes. See “Preupdate Rule Set Category”.
Reopened Rule Set Category
Trigger automated actions on reopening a claim or exposure. See “Reopened Rule Set Category”.
Segmentation Rule Set Category
Categorize a new claim or an exposure based on complexity, severity of damage, or other attributes. See “Segmentation Rule Set Category”.
Transaction Approval Rules Set Category
Determine if a user has the required authorization to submit a financial transaction. See “Transaction Approval Rule Set Category”.
Validation Rule Set Category
Check for missing information or invalid data on Address Book, Claim, Exposure, Matter, Policy and Transaction objects. See “Validation Rule Set Category”.
Workplan Rule Set Category
Add initial activities to a claim, exposure or matter as a checklist of work that various people need to do on the claim. See “Workplan Rule Set Category”.
