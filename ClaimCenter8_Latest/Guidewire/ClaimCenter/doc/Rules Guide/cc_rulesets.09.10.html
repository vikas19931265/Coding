Initial Reserve Rule Set Category

ClaimCenter uses Initial Reserve rules to create initial reserves on new exposures. The Initial Reserve rule category executes immediately after the Postsetup rules and has a specific purpose: to automate the creation of reserves for new exposures. If your enterprise does not automate reserve creation, you probably would not use this rule category.

Initial Reserve Rules

ClaimCenter uses these rules to determine the initial financial reserves to create for a new exposure. Each segment type has specific reserve requirements. For example, the amount of a reserve you establish for a moderate vehicle damage exposure is most likely less than the reserve you create for a serious bodily injury exposure. Whatever the amount of the reserve, the rule uses the createInitialReserve method to automate the creation of the reserve.
ClaimCenter does not associate a TransactionSet with any reserves added through these rules until it processes the new reserves. The most common approach for creating initial reserves is to write a rule or rule set for each exposure segment type.
In the following example, the rule automatically creates a reserve of $300,000 if the exposure coverage is of type PIP Death. The rule calls an enhancement method on Exposure.
Note: You can modify this rule to create a reserve of any amount for any kind of exposure.
Condition
return exposure.CoverageSubType.DisplayName == "PIP Death"
Action
print(actions.getRuleSet() + ":   " + actions.getRule() )
exposure.createInitialReserve( "claimcost", "death", 30000)
