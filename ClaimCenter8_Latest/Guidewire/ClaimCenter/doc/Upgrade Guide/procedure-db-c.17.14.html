Creating Extensions to Preserve Coverage Term Fields

The database upgrade removes the following fields from the CovTerm entity:
•
AggregationModel
•
RestrictionModel
•
Type
•
Value
•
ValueType
If you have data in any of these columns that you would like to preserve, add the columns as extensions to the CovTerm entity. If the database upgrade detects that any of these columns are defined in the data model, it does not delete the column.
If you have data defined for any of these columns, and you do not define an extension for the column, the database upgrade reports an error and stops.
If any of these coverage term fields have data for archived claims, the data is added as a note on the claim when the claim is restored.
To preserve coverage term fields
1.
Copy the CovTerm.etx file from the cc/config/extensions folder to configuration/config/extensions in the target configuration. If you already have CovTerm.etx in configuration/config/extensions, skip this step.
2.
Open CovTerm.etx in a text editor.
3.
Add a definition for each column that you want to preserve. The following example preserves each column that would otherwise be dropped in the upgrade:
 
<extension entityName="CovTerm">
  ...
  <column
    desc="The type of CovTerm"
    name="Type"
    type="shorttext"/>
  <column
    desc="Value (if direct) of this term."
    name="Value"
    type="decimal">
    <columnParam
      name="scale"
      value="4"/>
    <columnParam
      name="precision"
      value="20"/>
  </column>
  <column
    desc="The type of value in the Value field.  When imported from PolicyCenter, this captures the
          ValueType typecode."
    name="ValueType"
    type="shorttext"/>
  <column
    desc="The aggregation model to use for aggregate limits.  When imported from PolicyCenter, this
          captures the AggregationModel typecode."
    name="AggregationModel"
    type="shorttext"/>
  <column
    desc="The restriction model to use for coverage restriction.  When imported from PolicyCenter
          this captures the RestrictionModel typecode."
    name="RestrictionModel"
    type="shorttext"/>
</extension>
4.
Save and close CovTerm.etx.
