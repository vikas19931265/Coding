ClaimCenter Upgrade Guide : Upgrading from 6.0.x : Upgrading ClaimCenter from 6.0.x for ContactManager : Configuration File Changes

Configuration File Changes

The following table shows the files and classes used to configure ClaimCenter 6.0.x to work with ContactCenter on the left. If there are files or classes that replace them, they are shown on the right. You can use this table as a reference for the list of files you see in the configuration upgrade tool. The topic that follows this one walks you through updating these files and more.
IMPORTANT Before starting the configurations in this topic, Guidewire recommends that you first upgrade ContactManager and then refresh the ContactManager web services in the ClaimCenter web resource collection file AB800.wsc. See “Manually Upgrading ClaimCenter to Integrate with ContactManager”.

ab-to-cc-data-mapping.xml
An XML configuration file that maps properties, subentity names, and typelist names sent from ContactManager to ClaimCenter. Maps by change or omission: only fields that are different between the products or that are to be omitted are in this file. All others are sent automatically.
Do not use this file in 8.0.x. The 8.0.x files replace it.
•
ContactMapper.gs
A Gosu class that maps to XML the contact entity properties that you want send to ContactManager. It also maps from XML the contact properties that you want to receive from ContactManager. ContactManager has an equivalent file. If there are Contact subtypes, properties, arrays, or foreign keys that you have defined and want to send back and forth, you must map them in both ContactManager and ClaimCenter. You must map all properties that you want to send and receive. See “ContactMapper Class” in the Contact Management Guide.
•
CCNameMapper.gs
A ClaimCenter Gosu class that defines differences in entity and typecode names. See “Core Application ContactMapper Class” in the Contact Management Guide.
•
cc-to-ab-data-mapping.xml
An XML configuration file that maps properties, subentity names, and typelist names to send to ContactManager. Maps by change or omission: only fields that are different between the products or that are to be omitted are in this file. All others are received and handled automatically.
Do not use this file in 8.0.x. The 8.0.x files replace it.
•
ContactMapper.gs
Described previously. You must map all fields that you want to send and receive in this file.
•
CCNameMapper.gs
Described previously. A ClaimCenter Gosu class that defines differences in entity and typecode names.
•
search-filter.xml
An XML configuration file that controls the list of fields returned from ContactManager for search.
Do not use this file in 8.0.x. The 8.0.x files replace it.
•
ContactSearchResultMapper.gs
A ClaimCenter Gosu class that maps the results returned from a search sent to ContactManager. The class maps the fields to a Contact entity or subentity. It uses the ContactManager Web API ABContactAPISearchResult.
For more information, see “ClaimCenter Support for Contact Searches” in the Contact Management Guide.
•
ContactSearchCriteria entity
An entity that determines which contact fields are used for a local ClaimCenter contact search. Continue to use this file. Also, if you have added entries to it that you want to use in ContactManager search, you must update the 8.0.x files on the right.
•
ContactSearchCriteria entity

ContactSearchMapper.gs
A ClaimCenter Gosu class that maps search objects to a Gosu class instance that ClaimCenter sends to ContactManager.
For more information, see “ClaimCenter Support for Contact Searches” in the Contact Management Guide.
•
contact-sync-config.xml
Determines the contact properties that are included in the fingerprint used to identify a contact for synchronization. Defines the contact properties to be ignored when checking to see if a contact is synchronized between ClaimCenter and ContactManager. Additionally, controls the relationships that are synchronized or ignored when transferring contacts between ClaimCenter and ContactManager.
withAffectsSync(false)
Use this method, available in the ClaimCenter ContactMapper class, to exclude fields of Contact entities and subentities from the synchronization check.
RelationshipSyncConfig.gs
Replaces the XML file, which was visible in PolicyCenter and BillingCenter even though those core applications did not use it. Now available only for ClaimCenter, this Gosu class provides the same functionality as the XML file through its includeRelationship and excludeRelationship methods.
Package name – gw.plugin.contact.ab800
See “Synchronizing ClaimCenter Contact Fields” in the Contact Management Guide.
