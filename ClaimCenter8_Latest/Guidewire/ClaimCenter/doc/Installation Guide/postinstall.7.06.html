Setting Up Free-text Search for WebLogic

ClaimCenter free-text search requires that you install an instance of WebLogic separate from the instance that runs your ClaimCenter application. In a production environment, Guidewire requires that you set up the separate WebSphere instance on a host separate from the one that hosts your ClaimCenter application.
To set up free-text search for WebLogic
1.
Create a WebLogic domain/instance running under JDK 1.7.
2.
On the host where the WebLogic instance for free-text search resides, create an installation directory to use as the Guidewire Solr home directory. For example:
•
On Unix – /opt/gwsolr
•
On Windows – C:\opt\gwsolr
This guide uses /opt/gwsolr as the directory name.
3.
Create a new opt/gwsolr/cc subdirectory. In that directory, extract the file from your ClaimCenter application files:
    ClaimCenter/solr/cc-gwsolr.zip
4.
Within the autodeploy directory in the WebLogic instance, create a subdirectory cc-gwsolr. In that subdirectory, extract the contents of the cc-gwsolr.war.
5.
Check the web application deployment descriptor in autodeploy/cc-gwsolr/WEB-INF/web.xml for the definition of the solr.home environment variable:
 
    <env-entry>
      <env-entry-name>solr/home</env-entry-name>
      <env-entry-type>java.lang.String</env-entry-type>
      <env-entry-value>/opt/gwsolr/cc/solr</env-entry-value>
    </env-entry>
If you installed the ZIP file in a directory other than /opt/gwsolr, you must now change the path in the <env-entry-value> XML element to match the actual install directory path.
6.
Start the WebLogic instance.
7.
If you use SolrCloud, specify the appropriate Zookeeper arguments in the JAVA_OPTIONS environment variable. You can set the Zookeeper arguments by using a Unix shell script or a Windows batch file located in the domain base directory:
•
On Unix
 
        export JAVA_OPTIONS="-DzkHost=localhost:2181 -DnumShards=1 -DSTOP.PORT=7983 
             -DSTOP.KEY=key -Dbootstrap_conf=true" # use this line for first instance
 
        #export JAVA_OPTIONS="-DzkHost=localhost:2181 -DhostPort=8984" # this line adds an instance
 
        ./startWebLogic.sh
Alternatively, set the JAVA_OPTIONS environment variable by editing the setDomainEnv.sh script.
•
On Windows
8.
Proceed next to Setting Up the Free-text Batch Load Command.
