Catastrophe Search and Heat Map

ClaimCenter 7.0 includes the following new features and changes to catastrophe search and heat map:
•
“Catastrophe Search Page”
•
“Catastrophe Heat Map”
Note: Catastrophe search and heat maps in Guidewire ClaimCenter uses the Microsoft Bing mapping service in the base configuration. Use of other map services is possible through integration.

Catastrophe Search Page

All claims associated with a catastrophe are available for search and display on the ClaimCenter Catastrophe Search page for users with the View Catastrophes permission. This page can also be accessed from the Administration → Catastrophes → Catastrophe Details page.
The search mechanism on the ClaimCenter Catastrophe Search page is based on the catastrophe heat map. This means that catastrophe search uses geographic location as its main search criteria.
Besides providing a click-and-drag geographical search mechanism, the catastrophe heat map enables the user to visualize:
•
Where catastrophe losses are greatest.
•
Which claims reporting period was the most intense and where.
•
The status of claims in different regions within the catastrophe zone.
•
The geographic and numerical distribution of claims among ClaimCenter user groups.
See also
•
“Catastrophe Dashboard” in the Application Guide

Catastrophe Heat Map

In the base product, the catastrophe heat map distributes dots on an overlay covering a geographic area. The color of each dot represents a specific number of loss locations for claims entered against a particular catastrophe at that geographic location.
The catastrophe heat map can be configured to:
•
Show any other type of claims information for a geographic location, for instance, total loss amount.
•
Show several maps on a page with varying object information.
•
Retain the last state of any particular map so that the user can navigate away from the map and come back to it as they left it.
