Enabling Catastrophe Search and Heat Maps

Before you can see the Catastrophe Search page and its catastrophe heap map, you must enable the heat map facility as follows:
1.
Decide which geocoding service you want to use, and then configure a geocoding plugin to work with that service. If you use the Microsoft Bing Maps service, you can use the default geocoding plugin supplied with ClaimCenter. For more information on using and configuring another geocoding service, see “Geographic Data Integration” in the Integration Guide.
2.
Enable the geocoding plugin as described in “Configuring Geocoding”.
3.
License the Bing Maps Ajax Control from Microsoft. ClaimCenter uses version 6.3. Use the credential that you obtain from this license in the next step.
4.
Edit the ClaimCenter config.xml file to change the value of the heat map parameters:
a.
Open a command prompt to ClaimCenter/bin.
b.
Launch Guidewire Studio by running the following command from ClaimCenter/bin:
gwcc studio
c.
Double-click to open configuration → config → config.xml.
d.
Locate the following parameters and set them as the following example shows:
 
  <param name="HeatMapServiceTemplate" value="gw.api.heatmap.BingMap"/>
  <param name="HeatMapCredential" value="CREDENTIAL"/>
Set CREDENTIAL to the value that you obtained when you licensed the Bing Maps Ajax Control in step 3.
e.
Save your changes.
5.
If you have PolicyCenter integrated, there is a batch process, CatastrophePolicyLocationDownload, that you must enable. The batch process retrieves policy locations for a specific geographic area of interest from the policy system. The Catastrophe Heat Map shows retrieved policies as policy locations if the policy system geocoded the addresses.
To enable this batch process:
a.
Launch Guidewire Studio by running the following command from ClaimCenter/bin:
  gwcc studio
b.
Double-click to open configuration → config → scheduler → scheduler-config.xml.
c.
Remove any comments around the CatastrophePolicyLocationDownload configuration parameter. The following entry schedules the batch process to run every day at 2 a.m.:
 
  <ProcessSchedule process="CatastrophePolicyLocationDownload">
    <CronSchedule hours="2"/>
  </ProcessSchedule>
For more information on scheduling batch processes, see “Scheduling Batch Processes and Work Queues”.
For more information on this batch process, see “Catastrophe Policy Location Download”.
6.
For a development environment, restart the server using the gwcc dev-stop and gwcc dev-start commands. You can optionally load the catastrophe sample data supplied with ClaimCenter. This enables you to see the Catastrophe Search page and heat maps before you have loaded your own claims and catastrophe data. See “Installing Sample Data” in the Installation Guide.
For a production environment, stop the server. Then, repackage and redeploy the WAR or EAR file and restart the server. See “Deploying ClaimCenter to the Application Server” in the Installation Guide.
