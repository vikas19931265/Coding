EffDatedKey  Derived property returning gw.pl.persistence.core.effdate.EffDatedKey     (virtual property)?
Get an effective date independent key for this bean. This key will include the BranchId and the FixedId. This is the key for the "logical" bean.

An EffDatedKey only really applies to EffDateds as an EffDatedBranch is the same throughout its effective range (or its period range) so there is no difference between the "logical" bean and the actual bean.

Promoted  Derived property returning boolean     (virtual property)?
Check if this bean or branch has been promoted. A promoted bean is also locked and cannot be changed.

An EffDated is never promoted independent of its related EffDatedBranch, meaning if the branch is promoted every bean in the graph will be promoted, and if the branch is not promoted no bean in the graph can be promoted.

Date which this bean is "sliced" or viewed. If you traverse from this bean to another bean or beans via a link or array, only beans that are effective at this slice date will be returned. The beans that are returned will be sliced at this date as well. If NULL then the bean is in "window" mode.

In most cases a bean will be sliced, window mode is used in special cases.

VersionList  Derived property returning gw.pl.persistence.core.effdate.EffDatedVersionList     (virtual property)?
Get the version list for this bean. An EffDatedVersionList is a representation of a logical bean through its effective range. For example, if a bean has changes at three separate effective dates then the version list will comprise 3 versions of the bean, one for each of these dates.

You can use an EffDatedVersionList to traverse from this bean to other linked beans in window mode. Recall that in window mode you cannot traverse links directly from the bean.

Although an EffDatedBranch only has a single version throughout its effective range you can still get its EffDatedVersionList in order to navigate in window mode from the branch to its related EffDateds.
