Message Generator

This page is designed to let students rapidly create an instance of an event-aware entity, thereby triggering the creation of a message.

How Does It Work?

Whenever this page is rendered, TrainingApp auto-generates a name for a new instance of the MessageGenerator entity. This appears in the Name field. You can modify the name if you wish, but you do not need to. Clicking either "Create..." button creates a MessageGenerator instance using the name in the Name field. The Message Generator destination (ID 21) listens for events in which a MessageGenerator instance is created.

When you click the "Create and Send Only" button:

  1. An instance of MessageGenerator is created and committed. The AutoAckMessage field is set to false.

  2. The Message Generator destination creates a MessageContext and passes it to the Event Fired rules.

  3. Event Fired rule MGEF1000 creates a message. The payload is the Name field of the MessageGenerator instance. The message is then committed to the Message table.

  4. From the Message table, the MessageGeneratorTransport plugin prints the payload the the console.

Messages created via the "Create and Send Only" button are in the Message table and can be viewed via the Message Table screen.

When you click the "Create, Send, and Acknowledge" button:

  1. An instance of MessageGenerator is created and committed. The AutoAckMessage field is set to true.

  2. The Message Generator destination creates a MessageContext and passed it to the Event Fired rules.

  3. Event Fired rule MGEF1000 creates a message. The payload is the Name field of the MessageGenerator instance. The message is then committed to the Message table.

  4. From the Message table, the MessageGeneratorTransport plugin prints the payload the the console.

  5. Because the AutoAckMessage field is true, the MessageGeneratorTransport plugin also acknowledges the message.

Messages created via the "Create, Send, and Acknowledge" button reside in the Message table for the short period of time between message creation and message sending. They can be viewed via the Message Table screen if you look for them immediately after creating them. Once sent and acknowledged, the message moves to the MessageHistory table and can be viewed via the MessageHistory Table screen.
