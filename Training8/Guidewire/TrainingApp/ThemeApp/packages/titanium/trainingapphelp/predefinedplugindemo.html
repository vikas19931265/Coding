Predefined Plugins Demo

What Is A Predefined Plugin?

Guidewire applications include plugins that can be grouped into three functional categories. A predefined plugin is a Gosu or Java class which implements a set of methods triggered by internal Guidewire code that relate to fundamental application behavior. (There are also messaging plugins, which are used to send asynchronous messages and potentially process replies to those messages, and startable plugins, which listen for and process incoming messages unrelated to any outbound Guidewire message.)

What Is The Predefined Plugin Demo?

TrainingApp has a predfined IExchangeRateSet plugin. This plugin is designed to retrieve a set of exchange rates for all currencies known to TrainingApp. The Predefined Plugin Demo lets you query for a new set of exchange rates and see the most recently retrieved rates.

How Do I Use The Demo?

Click the "Invoke ExchangeRateSet Plugin" button. This executes the plugin, which creates a new set of exchange rates and displays them on the screen. There are two ways that you can tell you are viewing new exchange rates:

  • The Retrieved On column shows the date and time the exchange rates were retrieved.

  • The base exchange rate (from USD to any other non-USD currency) increments by 1.

How Is This Demo Implemented?

The "Invoke ExchangeRateSet Plugin" button calls the gw.api.util.CurrencyUtil.invokeMarketExchangeRateSetPlugin() method, which executes the ExchangeRateSet plugin. In the TrainingApp base application, the class registered as the plugin is trainingapp.demo.plugin.ExchangeRatePlugin.

The ExchangeRatePlugin class does not attempt to retrieve any real exchange rates. It simply sets the exchange rates from the base currency (USD) to all non-base currencies to an integer value. This value increases by 1 every time the plugin is called, which simulates the change of exchange rates over time.

Is This How Exchange Rates Are Done In Real Applications?

Absolutely not. This demo was created to demonstrate simple predfined plugin functionality, not to replicate a real-world implementation of exchange rates. There are two significant ways in which this demo is not real-world:

  • The demo requires manual action for new exchange rates to be retrieved. In a real implementation, the exchange rates would be retrieved automatically and periodically, such as through a batch process.

  • The plugin uses a simple, sequentially incremented integer value for all base-to-non-base exchange rates. In a real implementation, the exchange rates would be determined by calling some outside system.
