ClaimCenter Configuration Guide : Data Model Configuration : The ClaimCenter Data Model : Data Object Subelements : <typekey>

<typekey>

The <typekey> element defines a field for which a typelist defines the values. Guidewire defines this element in the data model metadata files as the <typekey> XML subelement.
Note: For information on typelists, typekeys, and keyfilters, see “Working with Typelists”.
Attributes of <typekey>
The <typekey> element contains the following attributes.
 
<typekey> attribute               
Description
Default
columnName
Optional. If specified, ClaimCenter uses this value as the column name of the corresponding database column. If you do not specify a columnName value, then ClaimCenter uses the value of the name attribute for the database column name.
IMPORTANT All column names on a table must be unique within that table. Otherwise, Studio displays an error if you verify the resource and the application server fails to start.
None
createhistogram
Whether to create a histogram on the column during an update to the database statistics.
Note: It is possible to override this attribute on an existing column in an extension (*.etx) file using the <column-override> element. You can use the override to turn off an existing histogram or to create one that did not previously exist.
This change does not take effect during an upgrade. The change occurs only if you regenerate statistics for the affected table by using the Guidewire maintenance_tools command.
See also

“Working with Attribute Overrides”
•
“Configuring Database Statistics” in the System Administration Guide
•
“Maintenance Tools Command” in the System Administration Guide
False
default
The default value given to the field during new entity creation.
None
deprecated
If true, then ClaimCenter marks the typekey as deprecated in the Data Dictionary and places a Deprecated annotation on it in the Guidewire Studio API Reference.
If you deprecate a typekey, use the description attribute (desc) to explain why.
For more information, see “The deprecated Attribute”.
False
desc
A description of the purpose and use of the field.
None

Determines whether you can transmit this field as part of a SOAP call. This only applies to the old-style RPC-Encoded style SOAP web services. It does not apply to WS-I web services. This attribute has nothing to do with XML import or export or with staging table loading, which is often known as import.
IMPORTANT Do not set this to true on any new entities that you create.

Description

Specifies a keyfilter to use to filter the typelist. This element requires the <name> attribute.
This attribute defines a relative path, navigable through Gosu dot notation, to a physical data field. Each element in the path must be a data model field.
