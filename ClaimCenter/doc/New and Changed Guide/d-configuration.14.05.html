Changes to Configuring the ClaimCenter Check Wizard

The Check Wizard in previous releases of ClaimCenter depended on custom wizard widgets to perform special functionality. These custom widgets made the Check Wizard less configurable than other parts of the application.
In ClaimCenter 7.0, configuring the Check Wizard screens is more flexible than in previous releases. Guidewire has made the following changes:
•
Removed special Check Wizard widgets from PCF files, such as NormalCreateCheckWizard, and moved their functionality to the gw.api.financials.CheckWizardInfo class and its subtypes. The Check Wizard PCF files now use standard wizard widgets.
•
Kept critical functionality in Java in the class gw.api.financials.CheckWizardInfoBase, while moving more flexible aspects of functionality to Gosu subtypes of that class.
•
The most commonly configured aspects of Check validation have been moved to the class CheckValidationMethodsImpl. These validations are enforced both in the Check Wizard and when creating a check in rules that use the CheckCreator methods on Claim and Exposure.
•
Refactored the Check Wizard, including moving functionality from Java classes to Gosu classes to make it more configurable. These API changes are still evolving, and some of the new Gosu classes are still read-only.
You can modify the Check Wizard PCF files to provide defaults and add additional code. Your changes must be additive. Do not remove method calls or logic from the Check Wizard without consulting Guidewire.
For creating checks in rules and locations other than the user interface, use a CheckCreator available from Claim and Exposure. For more information, see “Creating Checks and Payments by Using CheckCreator” in the Configuration Guide.
 
 
 Priority:
