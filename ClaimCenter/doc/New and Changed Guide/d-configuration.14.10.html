Changes Related to ContactCenter

In ClaimCenter 7.0, Guidewire has made the following changes to support ContactCenter:
•
ContactCenter is now called ContactManager and is the system of record for contacts. For ContactManager new features and changes, see “New and Changed Features in ContactManager” in the Contact Management Guide.
•
Changes have been made to the files you use to configure ClaimCenter to integrate with ContactManager. If you have extended the Contact entity or its subtypes in any way, there are multiple files that you must update. See “Configuration File Changes” in the Upgrade Guide.
•
The Save to Claim and Addressbook button is visible only if the contact is linked and synchronized with ContactManager. Also, the relationships to be synchronized must be the same.
•
FindDuplicates searches only the contact subtype its subtypes, as defined in the FindDuplicates plugin. It does not compare parents of the current contact subtype as it did in previous releases. For example, if the subtype of the contact to check for duplicates is ABCompanyVendor, FindDuplicates does not look at its parent subtype, ABCompany. The default subtypes defined for FindDuplicates are:
•
ABCompanyVendor
•
ABCompany
•
ABPersonVendor
•
ABPerson
•
ABPlace
