Accounts and Accounts Special Handling

An account is a organization or person that might have one or more policies. The account might be associated with no policies, one policy, or many policies. This association is made by matching the Account.AccountNumber field in ClaimCenter with the policy’s account number field.
Accounts exist in ClaimCenter for two main purposes
•
Reporting
•
Special handling instructions
ClaimCenter 7.0 includes the following new features and changes to accounts and accounts special handling:
•
“New Account Manager Role”
•
“New Accounts Page”
•
“Special Handling”
•
“New Account Permissions”
•
“Account Entity and Policy Data Model Changes”

New Account Manager Role

The Account Manager role is new in ClaimCenter 7.0. The Account Manager can:
•
Create, edit, and delete accounts on the Administration → Accounts page.
•
Set up any automated notifications (such as email), activities, or instructions to selected accounts.
In the base configuration, this role has been added to user Super User.
You can find out more about this new role in “Account Manager Permissions” in the Application Guide.

New Accounts Page

The Administration → Accounts page can be accessed by any user with the Account Manager role, or with View Account permissions.
Any user with the Accounts Manager role or with Manage Account permissions can now manage accounts on the Administration → Accounts page.
Accounts management tasks include:
•
Adding basic account information
•
Deleting an account
•
Adding, modifying, or deleting:
•
Automated notifications
•
Automated activities
•
Other instructions
See also
•
“Working with Accounts” in the Application Guide

Special Handling

Account Managers and users with Manage Account permissions can now set up special handing instructions for accounts. These instructions usually come in the form of automated notifications, activities, and other instructions.
In the base configuration, ClaimCenter can send automated notifications and activities for certain claim indicator events such as:
•
The coverage is in question
•
Fatalities
•
Flag details
•
Large losses
•
The claim is in litigation
ClaimCenter can also send automated notifications and activities for certain financial events, such as when a claim reaches a specific total net incurred amount, net total paid, or total paid. You define the threshold amount.
For more information on special accounts handling, see “Special Handling” in the Application Guide.

Account Entity and Policy Data Model Changes

ClaimCenter has added the Account entity. This was done to accommodate special account handling.
Policy.AccountNumber has also been added. It can be indirectly accessed by the Account entity.
