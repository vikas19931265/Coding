Changes to FNOL Matcher Classes

The following changes happened in the FNOL Mapper tool:
•
In the exposure mapper (IExposureMapper.gs), the various uses of the getExposure method have now been renamed based on their more specific usage. For example, the vehicle incident version of this method is now called getExposureForAutoLossInfoAndVehicleIncident.
•
In the incident mapper (IIncidentMapper.gs), the various getIncidentFor... methods now take a Claim object as an additional first argument.
•
Additionally, the files AcordFNOLMapper.gs, AcordExposureMapper.gs, and AcordIncidentMapper.gs changed to accommodate these changes.
This change is not expected to require an upgrade step for typical customers. If you changed any of these files, you will need to merge in these changes.
