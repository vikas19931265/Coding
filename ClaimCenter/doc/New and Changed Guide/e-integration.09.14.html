Plugin Registry Changes in Studio

Due to larger changes in ClaimCenter Studio, the user interface changed for registering plugins. See “Using the Plugins Registry Editor” in the Configuration Guide.
In version 8.0, there are several options for registering code that implements a plugin interface, but the deployment options vary:
•
Gosu – a Gosu class
•
Java – a Java class. ClaimCenter uses the location of classes and libraries on disk to determine whether to use the legacy Java class loader or the ClaimCenter 8.0 Java class loader. For a complete reference, see “Where to Deploy Your Java Classes and Libraries”. For the 8.0 Java API, also see “Deploying Non-OSGi Java Classes and JARs” in the Integration Guide.
