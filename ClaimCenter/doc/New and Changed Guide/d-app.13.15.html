Changes to Reinsurance Editable Fields on Claim Status Page

Two fields on the Claim Status page have been used to manually mark whether a claim was reinsurance reportable and the reason that it was reinsurance reportable
•
Reinsurance Reportable?
•
Reinsurance Edit Reason
Now these fields are auto-populated when reinsurance treaties are applied—when an exposure is created on the claim. The value of the Reinsurance Edit Reason field changes as reserves and payments are created.
Any user with Edit reinsurance reportable permissions can make these changes. By default, the following roles have this permission incorporated and can edit these fields:
•
Superuser
•
Adjuster
•
Reinsurance Manager
