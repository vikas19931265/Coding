Financial Holds

The financial holds feature has been added to enable a carrier to record and track claims, but prevent any indemnity payments. Expense and other types of payments can still be made on claims with financial holds.
In the base product, claims are automatically marked for financial hold in three cases:
•
The coverage is in question.
•
The claim has been marked as incident only.
•
The associated policy is unverified.
The financial hold status is not stored in the database. Rather, each time a transaction against an exposure is created, ClaimCenter checks to see whether the claim has been flagged as a financial hold. If it has been flagged as a financial hold:
•
No indemnity payments or reserves can be created. That is, no claim cost reserves or payments can be initiated.
•
Recoveries and recovery reserves can still be created.
•
Other reserves and payments, such as expenses, can be created, but the user is warned.
