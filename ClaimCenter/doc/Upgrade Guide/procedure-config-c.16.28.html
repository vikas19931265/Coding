Changes to HistoryType Typelist

In ClaimCenter 7.0.1 Guidewire has removed the retrieved typecode from the HistoryType typelist. The display name for the restored typecode has been changed to Retrieved. You might encounter the following UpgradeException error when running the database upgrade:
com.guidewire.pl.system.exception.UpgradeException: [
### reference(s) found from cc_history.Type to orphaned typecodes from HistoryType. You need to restore
the values to the typelist or update the values in cc_history.Type before upgrading. A query to find
these references is: SELECT t.ID, t.Type, tl.TYPECODE FROM cc_history t, cctl_historytype tl WHERE
t.Type = tl.ID AND tl.TYPECODE = 'retrieved']
If the database upgrade reports this error, add the retrieved typecode back to the HistoryType typelist and attempt to run the database upgrade again. The HistoryType typelist is defined in config/metadata/HistoryType.tix. The retrieved typecode definition is as follows:
<typecode
  code="retrieved"
  desc="Claim was retrieved from the queue"
  name="Retrieved from queue"/>
