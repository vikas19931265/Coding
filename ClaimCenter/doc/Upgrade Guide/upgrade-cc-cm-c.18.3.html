Manually Upgrading ClaimCenter to Integrate with ContactManager

This topic describes tasks you might have to perform to complete a ClaimCenter 6.0.x upgrade when you have ContactCenter installed.
Prior to performing the tasks in this topic, do the following:
1.
Run the Configuration Upgrade tool and perform the automatic upgrade for the ClaimCenter configuration. See “Upgrading the ClaimCenter 6.0.x Configuration”. Do not make changes yet to the files listed in “Configuration File Changes”. You make those changes later as described in this topic.
2.
Run the Database Upgrade tool to upgrade for the ClaimCenter database. See“Upgrading the ClaimCenter 6.0.x Database”. Among the other upgrades this tool performs, it makes changes to the tags for all your contacts.
3.
You can perform any manual configuration upgrades except those related to files listed in the previous topic, “Configuration File Changes”. Before making those changes, wait until you configure ContactManager, regenerate its SOAP API, and refresh that API in ClaimCenter studio, as described in the steps that follow.
4.
Manually configure ContactManager. See “Upgrading ContactManager from 6.0.x”.
5.
Integrate ClaimCenter and ContactManager as described at “Integrating ContactManager with Guidewire Core Applications” in the Contact Management Guide.
6.
In ClaimCenter, refresh the web service collection for the ContactManager ABContactAPIweb service to pick up the ContactManager changes.
a.
Start ContactManager. Open a command window, navigate to ContactManager/bin, and enter the following command:
    gwab dev-start
b.
Start ClaimCenter Studio. Open a command window and navigate to ClaimCenter/bin, and then enter the following command:
    gwcc studio
c.
In the ClaimCenter Studio Project window, navigate to configuration → gsrc and then to wsi.remote.gw.webservice.ab.ab800.wsc.
Note: The following steps describe specific settings in the editor for web services collections. For general information, see “Using the WS-I Web Service Editor” in the Configuration Guide,
d.
Double-click the ab800.wsc web services collection to open the editor, and then, in the editor, click the following resource in the Resources pane:
    ${ab}/ws/gw/webservice/ab/ab800/abcontactapi/ABContactAPI?wsdl
The ${ab} variable in this path corresponds to the product name="ab" entry in the suite-config.xml file that specifies the URL for ContactManager.
e.
With ContactManager running, click Fetch Updates to get the latest updates to ABContactAPI.

Mapping Your Contact Extensions

If you have made extensions to the Contact entity, such as new subtypes or fields, the files you use to map these extensions to ContactManager have changed and require updating. For an overview of these mapping files, see “Working with Contact Mapping Files” in the Contact Management Guide. To update the new files:
1.
Examine the ClaimCenter 6.0.x files cc-to-ab-data-mapping.xml and ab-to-cc-data-mapping.xml.
In ClaimCenter 6.0.x Studio in the Resources pane, navigate to configuration → Other Resources, and then click each file to open it in an editor.
2.
Update the Gosu class CCNameMapper with the names of any extended subentities and any typelists whose names differ between ClaimCenter and ContactManager.
In ClaimCenter 8.0.x Studio press CTRL+N and enter CCNameMapper to find this class, and then double-click the class name to open it in the editor.
3.
Update the ContactMapper class in ClaimCenter and ContactManager to map new entity fields, foreign keys, and arrays between ClaimCenter and ContactManager.
If you have extended a Contact entity or subentity with a new field or you have added a new Contact subentity, you must update the ClaimCenter class gw.contactmapper.ab800.ContactMapper. This class maps entities to XML to send to ContactManager, and it maps them from the XML sent from ContactManager. You must map all fields that you want to receive from ContactManager or send to it.
For more information, see “ContactMapper Class” in the Contact Management Guide.
4.
As described in “Configuration File Changes”, the ClaimCenter contact-sync-config.xml file is no longer being used. Its functionality has been replaced by the withAffectsSync method in ContactMapper and the RelationshipSyncConfig class. If you use contact-sync-config.xml to exclude fields or relationships, you must port your settings.
See “Synchronizing ClaimCenter Contact Fields” in the Contact Management Guide.

Working with the New Search Files

This topic applies if you have added your own entries to ContactSearchCriteria.etx or search-filter.xml. You might have done so if you extended a Contact entity or subentity and you wanted to add new fields to Search. If you have added your own entries to these files, you must also add them to two new Gosu files. These file are:
•
gw.plugin.contact.ab800.ContactSearchMapper – Defines search criteria sent to ContactManager. If you added entries to ContactSearchCriteria, you must also add them to ContactSearchMapper. ClaimCenter uses both files.
•
gw.plugin.contact.ab800.ContactSearchResultMapper – Defines the search results received from ContactManager to display in the search results screen. This file replaces search-filter.xml. If you have added settings to search-filter.xml, you must enter those settings in this file.
