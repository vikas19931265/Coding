ClaimCenter Upgrade Guide : Upgrading from 6.0.x : Upgrading ContactManager from 6.0.x : Configuration File Changes

Configuration File Changes

The following table shows on the left the files and classes used to configure ContactCenter 6.0.x to work with ClaimCenter. If there are files or classes that replace them, they are on the right. You can use this table as a reference for the list of files you see in the configuration upgrade tool.
 
6.0.x File
8.0.x File
•
Any ABContact extension
If you have added any properties to an ABContact entity or subentity or if you have added any new subentities, you might need to add them to the mapper class. You add the subentities and properties that you want to transfer to and receive from Guidewire core applications.
•
ContactMapper.gs
A Gosu class that maps to XML the contact entity properties that you want send to ClaimCenter. It also maps from XML the contact properties that you want to receive from ClaimCenter. ClaimCenter has an equivalent file. If you have ABContact subtypes, properties, arrays, or foreign keys that you have defined and want to send back and forth, map them in both ContactManager and ClaimCenter. You must map all properties that you want to send and receive.
For more informations, see “ContactMapper Class” in the Contact Management Guide.
•
IContactAPI
The web services API that ContactCenter provided to Guidewire applications for integrating with ContactCenter.
Do not use this file in 8.0.x. ABContactAPI replaces it.
•
ABContactAPI
The replacement for IContactAPI. A WS-I compliant SOAP service that provides methods allowing more flexibility in supporting different configurations.
For more information, see “ABContactAPI Web Service” in the Contact Management Guide.
•
definitive-match-config.xml
In previous releases, ContactManager used definitive matching to uniquely identify a contact. It was based on a set of configurable fields defined in this file. The set of fields for a contact constituted a unique ID for the contact, a MatchSetKey enforced at the database level with a unique index on the composite key. This definitive matching was used as the first step both in creating new contacts and in linking to contacts from Guidewire applications.
Do not use this file. If you have added your own entries to the file, you can implement them in ValidateABContactCreationPluginImpl.
•
ValidateABContactCreationPluginImpl
A plugin implementation that enables you to provide your own logic for creating new contacts. In the base implementation, all contacts are required to have a contact tag. The base implementation provides a set of more relaxed minimum creation requirements.
•
Note: The MatchSetKey column on ABContact has been deprecated.
•
For more information, see “ValidateABContactCreationPlugin Plugin Interface” in the Contact Management Guide.
•
potential-match-config.xml
Potential match searching in ContactCenter was based on elements defined in this file.
Do not use this file. If you have defined your own potential match logic in this file, you must reimplement it in Gosu in FindDuplicatesPlugin.
•
IFindDuplicatesPlugin
Attempts to find exact and potential matches for the subtype of ABContact that is passed to it. If no Duplicate Finder exists for the subtype, the plugin traverses the ABContact tree to find a parent type it can use in the search.
The plugin uses a set of Duplicate Finder classes, which define potential and exact match logic for a set of ABContact subtypes.
For more information, see “IFindDuplicatesPlugin Plugin Interface” in the Contact Management Guide.
•
AdditionalPrimaryABContactSearchCriteria in config.xml
Entry in config.xml that specifies that a Contact entity field was required in a search.
If you have added your own criteria, convert them to Gosu statements in ValidateABContactSearchCriteriaPluginImpl. Then remove these entries from config.xml.
•
ValidateABContactSearchCriteriaPluginImpl
A Gosu class that specifies required search fields.
For more information, see “Local ContactManager Search Criteria” in the Contact Management Guide.
•
ABContactSearchCriteria entity
An entity that determines which contact fields are used for a search. Continue to use this file. Also, if you have added entries to it, you must update the 8.0.x files.
•
ABContactSearchCriteria entity
•
ABContactAPISearchCriteria
A ContactManager Gosu class that specifies the search fields sent by core applications to ContactManager.
•
ABContactAPISearchResult
A ContactManager Gosu class that specifies the contents of search results returned to Guidewire applications.
See “ContactManager Support for Contact Searches” in the Contact Management Guide.
•
search-config.xml
An XML configuration file that determines which ABContact fields are used for a search in the ContactManager user interface.
Continue to use this file..
•
search-config.xml
If you added your own entries to this file in ContactCenter 6.0.x, accept the file in the configuration tool to copy it to the ContactManager 8.0.x configuration.
See “ContactManager Support for Contact Searches” in the Contact Management Guide.
