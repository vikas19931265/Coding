File and PCF Changes in ClaimCenter Related to ContactManager

Web Service Version Changes

ClaimCenter now uses wsi.remote.gw.webservice.ab.ab801.wsc to access the ContactManager web service ${ab}/ws/gw/webservice/ab/ab801/abcontactapi/ABContactAPI?wsdl. See “Step 1: Integrate ContactManager with ClaimCenter” in the Contact Management Guide.

Driver’s License State Now Uses Jurisdiction.ttx

When you are editing a contact who is a Person subtype in ClaimCenter or an ABPerson subtype in ContactManager, there is a Driver's License section on the edit screen. Both ClaimCenter and ContactManager have a version of this edit screen. The State field on this screen now uses the Jurisdiction.ttx file to populate the drop-down list for the field. If you want to change the contents of the drop-down list, edit the Jurisdiction.ttx file in Studio and add or remove values from the outgoing category driving_lic.
If you have extended either screen, you must merge your changes and use Jurisdiction.ttx.
For example, the ClaimCenter PCF file ContactBasicsDV.Person.pcf uses DriversLicenseInfoInputSet.pcf. The State field on this InputSet is a TypeKeyInput widget with value Person.LicenseState. This field’s filter property has the following code to filter values from Jurisdiction.ttx:
VALUE.hasCategory(JurisdictionType.TC_DRIVING_LIC)
The ContactManager example is similar. The PCF file ContactBasicsDV.ABPerson.pcf has an unnamed InputSet that has two driver’s license fields. The State field is a TypeKeyInput widget with value (contact as ABPerson.LicenseState. This field’s filter property has the following code to filter values from Jurisdiction.ttx:
VALUE.hasCategory(JurisdictionType.TC_DRIVING_LIC)
