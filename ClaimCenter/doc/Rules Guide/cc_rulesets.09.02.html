ClaimCenter Rules Guide : Gosu Business Rules : Rule Set Categories : Rule Set Summaries

Rule Set Summaries

ClaimCenter Studio contains the following sample rule sets:
 
Rule set category
Contains rules to
Approval Routing
Route approval activities for a transaction that requires approval to the appropriate user. See “Approval Routing Rule Set Category”.
Archive
Manage the claim archive process. See “Archive Rule Set Category”.
Assignment
Determine the responsible party for an activity, claim, exposure, matter, or service request. See “Assignment Rule Set Category”.
BulkInvoice Approval
Define the approval requirements for Bulk Invoices. See “BulkInvoice Approval Rule Set Category”.
Closed
Perform an action any time that you close an activity, claim, exposure, or matter—for example, sending a notification. See “Closed Rule Set Category”.
Event Message
Handle communication with integrated external applications. See “Messaging and Events” in the Integration Guide for information events and event messaging. See “Event Message Rule Set Category”.
Exception
Specify an action to take under certain circumstances. For example, if no one touches a claim for a certain number of days, then create a review activity. See “Exception Rule Set Category”.
Initial Reserve
Determine the initial financial reserves that to create for a new exposure. See “Initial Reserve Rule Set Category”.
Loaded
Trigger automatic actions any time that you load a new claim using the FNOL import interface. See “Loaded Rule Set Category”.
Postsetup
Trigger automatic actions any time that you add a new claim within ClaimCenter. These rules run after the Assignment and Workplan rules complete their execution. See “Postsetup Rule Set Category”.
Presetup
Trigger automatic actions any time that you add a new claim, exposure, matter or transaction within ClaimCenter. These rules run prior to the execution of rules in the Segmentation, Assignment, and Workplan rule set categories. See “Presetup Rule Set Category”.
Preupdate
Trigger automatic actions any time that a claim or other high-level entity changes. See “Preupdate Rule Set Category”.
Reopened
Trigger automated actions on reopening a claim or exposure. See “Reopened Rule Set Category”.
Segmentation
Categorize a new claim or an exposure based on complexity, severity of damage, or other attributes. See “Segmentation Rule Set Category”.
Transaction Approval
Verify that a user has the appropriate authorization to submit a financial transaction. See “Transaction Approval Rule Set Category”.
Validation
Check for missing information or invalid data on Address Book, Claim, Exposure, Matter, ServiceRequest, Policy and Transaction objects. See “Validation Rule Set Category”.
Workplan
Add initial activities to a claim, exposure or matter as a checklist of work that various people need to do on the claim. See “Workplan Rule Set Category”.
