Gosu Reference Guide : Query Builder APIs : Restricting Queries with Predicates on Fields : Predicate Methods Reference

Predicate Methods Reference

The following table lists the types of comparisons and matches you can make with methods on the query object.
 
Predicate method   
Arguments (Type)             
Description                                                                                      
compare
•
Column name (String)
•
Operation type (Relop)
•
Value (Object)
Compares a column to a value. For the operation type, pass one of the following values to represent the operation type:
•
Equals – Matches if the values are equal
•
NotEquals – Matches if the values are not equal
•
LessThan – Matches if the row’s value for that column is less than the value passed to the compare method.
•
LessThanOrEquals – Matches if the row’s value for that column is less than or equal to the value passed to the compare method.
•
GreaterThan – Matches if the row’s value for that column is greater than the value passed to the compare method.
•
GreaterThanOrEquals – Matches if the row’s value for that column is greater than or equal to the value passed to the compare method.
Pass these values without quote symbols around them. These names are values in the Relop enumeration.
For the value object, you can use numeric types, String types, ClaimCenter entities, keys, or typekeys. For String values, the comparison is case-sensitive.
Example of a simple equals comparison:
query.compare("Priority", Equals, 5)
Example of a simple less than or equal to comparison:
query.compare("Priority", LessThanOrEquals, 5)
To compare the value to the value in another column, generate a column reference and pass that instead. See “Comparing Column Values with Each Other” for extended discussion on this subject.
You can use algebraic functions that evaluate to an expression that can be evaluated at run time to be the appropriate type. For example:
var prefix = "abc:"      // string variable...
var recordNumber = "1234"
query.compare("PublicID", Equals, prefix + recordNumber)
Or combine a column reference and algebraic functions:
query.compare("Priority", Equals,
       DBFunction.Expr({query.getColumnRef("OldPriority"),
                "+", "10"}))
compareIgnoreCase
•
Column name (String)
•
Operation type (Relop)
•
Value (Object)
Compares a character column to a character value while ignoring uppercase and lowercase variations. For example, if the following comparison succeeds:
query.compare("Name", Equals, "Acme")
The following comparison also succeeds:
query.compareIgnoreCase("Name", Equals, "ACME")
startsWith
•
Column name (String)
•
Substring value (String)
•
Ignore case (Boolean)
Checks whether the value in that column for each row starts with a specific substring. For example, if the substring is "jo", it will match the value "john” and "joke" but not the values "j" or "jar". If you pass true to the Boolean argument (the third argument), Gosu ignores case differences in its comparison. For example:
query.startsWith("FirstName", "jo", true /* ignore case */)
Note: If you choose case insensitivity, Gosu generates an SQL LOWER function to implement the comparison predicate. However, if the data model definition of the column specifies the supportsLinguisticSearch attribute set to true, Gosu uses the denormalized version of the column, instead.
contains
•
Column name (String)
•
Contains value (String)
•
Ignore case (Boolean)
Checks whether the value in that column for each row contains a specific substring. For example, if the substring is "jo", it will match the value "anjoy” and "job" but not the values "yo" or "ji". If you pass true to the final argument, Gosu ignores case differences in its comparison. For example:
query.contains("FirstName", "jo", true /* ignore case */)
compare
•
Range (a Range subclass)
Compares the row value for that column to a specified range. The Range class is a superclass (an abstract type) with several subclasses that implement different type of ranges. Refer to the following three rows which list the range subclasses.
compare
•
Number range (NumberRange)
If the column type is a number (its type extends from Number), use the range subclass NumberRange. The following code defines a range of numbers:
query.compare("Priority", new NumberRange(5,10))
To specify an unbounded range on the lower or upper end, pass null to the constructor as the first or second range argument but not both. You can pass null as a range argument regardless of null restrictions on the column.
compare
•
Date range (DBDateRange)
If the column type is a date, use the range subclass DBDateRange. It takes two dates and then a Boolean value that specifies whether to normalize the dates. To normalize a date is to set its time component to midnight of that day. The following code defines the date range January 1, 1990, to January 1, 2010.
query.compare("DateOfBirth",
        new DBDateRange(new DateTime(90,0,1),
               new DateTime(110,0,1), true))
To specify an unbounded range on the lower or upper end, pass null to the constructor as the first or second range argument but not both. You can pass null as a range argument regardless of null restrictions on the column.
compare
•
Value range. Use with text values or anything other than date or number. (ValueRange)
For a general value range including String comparisons, use the ValueRange subclass and specify the type of value in generics notation: ValueRange<Typename>. For example, the following code defines a range of String values to match:
query.compare("PublicID",
  new ValueRange<String>("abc:1","abc:99"))
To specify an unbounded range on the lower or upper end, pass null to the constructor as the first or second range argument but not both. You can pass null as a range argument regardless of null restrictions on the column.
between
•
Column name (String)
•
Start value (Object)
•
End value (Object)
Checks whether a value is between two values. This is functionally very similar to the compare method when used with a range argument. This method supports String values, date values, and number values, just as the range version method compare.
query.between("PublicID", "abc:01", "abc:99")
To specify an unbounded range on the lower or upper end, pass null as the first or second range argument but not both. You can pass null as a parameter regardless of null restrictions on the column.
compareIn
•
Column name (String)
•
List of values that could match the database row for that column (Object[])
Compares the value for this column for each row to a list of objects that you specify. If the column value for a row matches any of them, the query successfully matches that row. For example:
query.compareIn("PublicID", {"default_data:1",
        "default_data:3"})
compareNotIn
•
Column name (String)
•
List of values that could match the database row for that column (Object[])
Compares the value for this column for each row to a list of objects that you specify. If the column value for a row matches none of them, the query successfully matches that row. For example:
query.compareNotIn("PublicID", {"default_data:1",
        "default_data:3"})
subselect
See “Using Set Inclusion and Exclusion Predicates”.
Perform a join with another table and select a subset of the data by combining tables. For more information, see “Using Set Inclusion and Exclusion Predicates”.
or
Gosu block that contains a list of predicate methods applied to columns in the query.
Checks whether a value satisfies one or more predicate methods, such as compare, contains, and between. Only one of the predicate methods must evaluate to true for the item that contains the value to be included in the result. For more information, see “Combining Predicates with AND and OR Logic”
and
Gosu block that contains a list of predicate methods applied to columns in the query.
Checks whether a value satisfies a set of predicate methods, such as compare, contains, and between. All of the predicate methods must evaluate to true for the item that contains the value to be included in the result. For more information, see “Combining Predicates with AND and OR Logic”
