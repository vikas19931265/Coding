Observe General Gosu Naming Conventions

As a best practice, Guidewire recommends the following general naming conventions.
 
Language Element  
Naming Conventions
Examples
Variable names
Name variables in mixed case, with an initial lower case letter and a medial capital for each internal word. Name variables mnemonically so that someone reading your code easily remembers what they represent. Do not use single letters such as “x” for variable names, except for short-lived variables, such as loop counts.
nextPolicyNumber
firstName
recordFound
Function names
Compose function names in verb form. Name functions in mixed case, with an initial lower case letter and medial capitals for each internal word. Add the suffix _Ext to the ends of function names to avoid future naming conflicts if Guidewire adds or changes base functions.
getClaim_Ext()
getWageLossExposure_Ext()
Class name
Compose class names in noun form. Name classes in mixed case, with an initial upper case and medial capitals for each internal word. Add the suffix _Ext to the ends of class names to avoid future naming conflicts if Guidewire adds or changes base classes.
StringUtility_Ext
MathUtility_Ext
