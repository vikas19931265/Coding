Policy (cc_policy) (delegates to EventAware, Extractable, Retireable, Validatable)
Description
Policy, and associates it to the claim. There is a one-to-one relationship between claims and policies in the system. If multiple claims have been filed against the same actual policy, there will be a distinct Policy for every Claim in the system.
Entity: Policy
Attributes: Editable?, Exportable?, Extendable?, Final?, Keyed?, Loadable?, Sourceable?, Versionable?
Messaging Events: The application will generate these events for this entity:
  • PolicyChanged
  • PolicyRemoved
  • PolicyAdded
Integration programmers can write Event Fired rules to perform actions on these events. Refer to the Integration Guide for the complete list of events for all entities. Remember to configure a new or existing messaging destination in Studio to listen for desired events.

PrimaryLocation  Derived property returning PolicyLocation     (virtual property)?
Returns the "primary location" on the policy, if any. The rules are a little complex, but can be configured if necessary, by changing the logic in this property:

  • The policy locations are sorted by location number
  • If a location has the PrimaryLocation bit set then that location is returned. If multiple locations have the bit set then the one with the lowest location number is used
  • If no location has the PrimaryLocation bit set then the first location (as ordered by location number) is returned
Most homeowners policies will just have a single PolicyLocation in any case. Note that location numbers are strings so the sorting is alphanumeric, not numeric.
